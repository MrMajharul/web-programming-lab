What are HTML Elements?

HTML elements are the building blocks of HTML pages. They consist of a start tag, content, and an end tag. Elements can contain other elements and attributes.

Element Categories

  • Block Elements: Take up full width and start on new line
  • Inline Elements: Take only necessary width and don't start new line
  • Self-closing Elements: Don't have closing tag
  • Semantic Elements: Have specific meaning (HTML5)
  • Form Elements: Used for user input and interaction

üì¶ Block vs Inline Elements

Block Elements

Block elements take up the full width available and start on a new line.

Element Purpose Example
<div> Generic container <div>Content</div>
<p> Paragraph <p>Text</p>
<h1>-<h6> Headings <h1>Title</h1>
<ul>, <ol> Lists <ul><li>Item</li></ul>
<table> Table <table><tr><td>Cell</td></tr></table>

Inline Elements

Inline elements take only the necessary width and don't start a new line.

Element Purpose Example
<span> Generic inline container <span>Text</span>
<a> Link <a href="#">Link</a>
<strong> Strong text <strong>Bold</strong>
<em> Emphasized text <em>Italic</em>
<img> Image (self-closing) <img src="image.jpg">

üì¶ Container Elements

<div> - Division

Generic block-level container for grouping elements.

<div class="container"> <p>Content</p> </div>
<span> - Span

Generic inline container for grouping elements.

<span class="highlight">Text</span>
<section> - Section

Semantic container for document sections (HTML5).

<section> <h2>Section Title</h2> <p>Content</p> </section>
<article> - Article

Semantic container for independent content (HTML5).

<article> <h1>Article Title</h1> <p>Article content</p> </article>
<aside> - Aside

Semantic container for sidebar content (HTML5).

<aside> <h3>Sidebar</h3> <p>Related content</p> </aside>
<header> - Header

Semantic container for page/section header (HTML5).

<header> <h1>Page Title</h1> <nav>Navigation</nav> </header>

‚úÖ Nesting Best Practices:

  • Always close tags in reverse order of opening
  • Don't put block elements inside inline elements

  • Use indentation to show nesting levels
